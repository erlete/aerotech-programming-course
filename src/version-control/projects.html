Introduction

Projects are excellent cross-project organization tools. They are schemas that can be used to organize issues and pull requests from multiple repositories into a single board. They can be used to track progress of issues and PRs, assign them to specific people, label and filter them, create new ones, close them, etc.

Projects as development headquarters

Understanding how to properly combine the functionalities of Issues, Discussions, Pull Requests and Projects can be the key to efficiently managing a project. If you ever find yourself in a situation where you don't know how to organize your work, try to follow these simple steps:
  1. Create a project for the repository you are working on
  2. Create issues describing future implementations
  3. Create branches for each issue as required due to the complexity of the implementation
  4. Create pull requests for each branch as development progresses
  5. Optionally, create discussions for pull requests that require further planning
This way, you will have a centralized place where you can track the progress of your work, discuss it with your team, and plan future implementations.
