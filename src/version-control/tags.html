Introduction

Tags are a way to mark specific commits in a repository. They are used to signify important milestones in a project, such as a release. Tags are immutable, meaning that they cannot be changed once they are created. They allow developers to offer a stable (release) version of their software to users. This is done by developing and testing the software until it reaches a near-flawless state. Then, a tag is created to mark the commit that contains the stable version of the software. As you can imagine, Python 3.11 is itself a tag that marks a commit in the Python repository.

While there are plenty of tag naming conventions, the most common one is semantic versioning. This convention consists of three numbers separated by dots, such as 1.0.0. The first number represents the major version, the second the minor version, and the third the patch version. The major version is incremented when there are breaking changes in the software, the minor version is incremented when there are new features, and the patch version is incremented when there are bug fixes. Furthermore, it is recommended to use a leading "v" character to signify that the tag is a version number (e.g., v1.0.0).
