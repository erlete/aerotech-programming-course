Introduction

Issues are the preferred method for problem reports, enhancement proposals or just development organization. They are petitions that can be created using the issues' interface. Each issue is composed of a title and a description. It is important to be clear while writing requests in these fields, since they have to be understandable to the rest of developers.

Classifiers

Both issues and pull requests can be classified through the following fields:
  • Assignees: people that are responsible for the resolution of the issue
  • Labels: additional classifiers for the issue
  • Projects: organization schemas that include the issue
  • Milestone: development goal that the issue contributes to
  • Development: list of linked PRs that aim to resolve the issue
