Introduction

A fork is a copy of a repository that is independent from the original one. Forks are usually used to propose changes to the original repository, which can then be merged into it. They are also used to create new repositories based on an existing one for custom functionalities' implementation. A key difference between a repository fork and a clone is that a fork keeps the Git history of the original repository, while a clone does not (while also giving credit to the original repository author).

Branches from a forked repository are referenced as username:branch-name, where username is the username of the user who forked the repository. This way, external contributions to a repository can be easily identified. For example, if a user named lkubi forks the aerotech-programming-course, which contains a branch named presentation-rework, the branch from the forked repository will be referenced as lkubi:presentation-rework.
