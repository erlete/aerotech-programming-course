Introducción

Las acciones son la clave para la automatización de proyectos y el desarrollo constante. Son scripts de código que se ejecutan ante eventos específicos durante el proceso de desarrollo (por ejemplo, un commit que se ha enviado al repositorio remoto, una rama que se ha fusionado con otra, se ha creado un PR, etc.).
Dado que las acciones se ejecutan en máquinas virtuales remotas, se pueden usar para construir, probar, empaquetar y desplegar tus proyectos con un esfuerzo mínimo. También se pueden usar para automatizar tareas repetitivas que de otro modo se harían manualmente, como formateo de código, linting, etc.

Acciones y PRs: la combinación perfecta

Las acciones se vuelven extremadamente útiles cuando se usan junto con reglas de protección de ramas, ya que permiten a los mantenedores del repositorio hacer cumplir ciertas reglas (por ejemplo, formateo de código, linting, etc.) antes de fusionar un PR en una rama protegida. Esto puede mejorar en gran medida la calidad del código y el proceso de desarrollo en general, al tiempo que reduce la cantidad de posibles errores gracias a la implementación de pruebas de código automáticas.
