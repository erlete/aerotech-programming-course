Introduction

Actions are the key for project automation and constant development. They are scripts of code that are executed upon specific events during the development process (e.g. a commit that has been pushed to the remote repository, a branch that has been merged into another one, a PR has been created, etc.).
Since Actions are executed in remote virtual machines, they can be used to build, test, package and deploy your projects with minimal effort. They can also be used to automate repetitive tasks that would otherwise be done manually, such as code formatting, linting, etc.

Actions and PRs: the perfect match

Actions become extremely useful when used along branch protection rules, since they allow repository maintainers to enforce certain rules (e.g. code formatting, linting, etc.) before merging a PR into a protected branch. This can greatly improve the quality of the codebase and the overall development process, while also reducing the amount of potential bugs thanks to the implementation of automatic code testing.
