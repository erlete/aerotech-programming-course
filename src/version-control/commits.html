Introduction

Commits are the core of Git. They are the way Git keeps track of the changes that happen to your files. A commit is a snapshot of your project at a certain point in time. It can contain one or more changes to your files, and it is identified by a unique hash. Furthermore, commits contain a title and a description. Titles are mandatory and should not exceed 50 characters. Descriptions are optional and can be as long as they need to be.

Why are repositories important?

Since the contents of a commit can be selectively applied to the version control process, commits offer the possibility of keeping track of small, independent changes (e.g. a typo fix, a missing semicolon, etc.) or complex ones (e.g. website UI redesign, new feature, etc.). This depends solely on the developer's criteria.

Furthermore, selective commits are very useful when working on unstable features, since if something breaks, it is easy to identify the commit that caused the issue and revert the state of the project to the previous one.
