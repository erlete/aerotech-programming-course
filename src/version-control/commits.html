Introduction

Commits are the core of Git. They are the way Git keeps track of the changes that happen to your files. A commit is a snapshot of your project at a certain point in time. It can contain one or more changes to your files, and it is identified by a unique hash. Furthermore, commits contain a summary (mandatory) and a description (optional). The summary should never exceed 50 characters, whereas descriptions can be as long as required.

It is always a good practice to maxify the amount of information given in a commit (summary abd description), while minimizing the amount of changes. This is due to the fact that having multiple, well-described changes on the project makes it easier to revert it to a previous state, or to identify the commit that caused a bug.
