Introduction

Pull Requests (PRs) are petitions to merge a branch into another branch when the user does not have sufficient rights to perform a direct commit. They can be used to create a halted state between development and implementation, which is useful when there are lots of changes that require exhaustive revisions. PRs are also useful to discuss changes before they are implemented, since they can be commented and reviewed by other developers.

Issues vs Pull Requests

Issues and PRs are similar in the sense that they are both used to discuss changes before they are implemented. However, they are different in the sense that issues are used to expose problems or potential implementations, while PRs are used to discuss solutions, including proposed changes. In other words, issues represent what the problem or suggestion is, whereas PRs represent how the problem or suggestion should be solved.
